## 2024-12-06

### 後端開發
1. 創建了任務（Task）模型
   - 包含基本字段：id, title, description, status, priority, dueDate, userId
   - 建立了與用戶（User）的關聯關係
2. 實現了任務的CRUD API
   - GET /api/tasks - 獲取所有任務
   - GET /api/tasks/:id - 獲取單個任務
   - POST /api/tasks - 創建任務
   - PUT /api/tasks/:id - 更新任務
   - DELETE /api/tasks/:id - 刪除任務
3. 添加了任務相關的路由和控制器
4. 完成了API的基本測試
