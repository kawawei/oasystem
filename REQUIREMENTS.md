# OA System 需求文檔

## 基礎指令規範
1. 每次更新代碼前需要提醒停止服務器
2. 每次完成一個模塊需要提醒進行 git 提交
3. 提供完整的 git 命令
4. 代碼更新需要逐步進行，不能跳步驟
5. 所有響應使用中文

## Git 提交規範
提交信息格式：`type: description`

類型（type）：
- feat: 新功能
- fix: 修復錯誤
- docs: 文檔更新
- style: 代碼格式調整
- refactor: 代碼重構
- test: 添加測試
- chore: 構建過程或輔助工具的變動

## 開發流程規範
1. 停止服務器
2. 安裝依賴
3. 更新代碼
4. 測試功能
5. Git 提交
6. 更新開發日誌

## 文件命名規範
1. 文件名使用 PascalCase（首字母大寫）
2. 類型定義文件使用 .d.ts 後綴
3. 測試文件使用 .test.ts 後綴

## 代碼風格規範
1. 使用 TypeScript 強類型
2. 所有函數必須有返回類型
3. 使用 async/await 處理異步
4. 統一使用單引號
5. 統一使用 UTF-8 編碼

## 錯誤處理規範
1. 所有錯誤信息使用中文
2. 統一使用自定義的 AppError 類
3. 錯誤需要記錄到日誌

## 開發順序
1. 基礎文件創建
2. 數據庫設計
3. 用戶認證
4. 核心功能 API
5. 即時通訊
6. 部署準備

## 專案概述
這是一個辦公自動化系統的後端服務，配合前端 Vue 3 + TypeScript 專案使用。

## 測試規範
1. API 測試方法
   ```bash
   # 健康檢查
   curl http://localhost:5000/api/health
   
   # 或使用瀏覽器訪問
   http://localhost:5000/api/health
   ```

2. 啟動服務器
   ```bash
   # 開發模式啟動
   cd server
   npm run dev
   
   # 生產模式啟動
   npm start
   ```

3. 測試流程
   - 啟動服務器
   - 執行 API 測試
   - 檢查錯誤處理
   - 驗證響應格式

4. 測試工具
   - curl: 命令行測試
   - Postman: API 測試工具
   - 瀏覽器: 直接訪問
   - Jest: 單元測試（待實現）

5. 測試檢查項目
   - 響應格式是否正確
   - 錯誤處理是否生效
   - 中文錯誤信息
   - HTTP 狀態碼

## 依賴安裝規範
1. 後端依賴安裝
   ```bash
   # 在 server 目錄下安裝
   cd server
   
   # 安裝主要依賴
   npm install <package-name>
   
   # 安裝開發依賴
   npm install -D <package-name>
   ```

2. 前端依賴安裝
   ```bash
   # 在專案根目錄下安裝
   npm install <package-name>
   ```

3. 安裝步驟
   - 停止服務器（如果正在運行）
   - 進入正確的目錄
   - 執行安裝命令
   - 確認 package.json 更新
   - 啟動服務器測試
